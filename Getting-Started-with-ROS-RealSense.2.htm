Intel RealSense Technology Integration to ROS (Robot Operating System)

What the package contains

You will find 3 folders:

f200_install
r200_install
realsense_dist

The first two contains all the files you need to install in order to access the cameras. In the 3rd you will find the specific ROS installation. Explanation on the installation process can be found in the next sections for each camera.

Please note that:

         IVCAM is also known as F200

         DS4 is also known as R200

Requirements:

Linux 12.04+

ROS Indigo

R200 (DS4) camera

* The ROS integration has been tested on a 64bit machine with Linux 14.04.01 and ROS Indigo.

Intel RealSense Technology R200 (DS4) Integration to ROS (Robot Operating System)

 

Requirements:

Linux 12.04+

ROS Indigo

R200 (DS4) camera

* The ROS integration has been tested on a 64bit machine with Linux 14.04.01 and ROS Indigo.

Getting Started:

Getting the camera to work on Linux

Please make sure that you have a working camera and that the software stack is installed properly. This can be checked by connecting the camera (to a USB3 port) and running one of the provided samples.

If this does not work, you should first fix this issue before continuing with the ROS integration.

In the package folder you will find r200_install folder.

Un-pack and follow the ReadMe file in order to install the R200 driver and API.

         The provided patch_ubuntu_uvc.sh has some changes to the script in the packed files that works with Ubuntu 14.04. !! Use this file instead in this case.

In the ds dependencies.txt you will find instructions how to download other dependences if you dont have them already.

After installation is complete, restart your machine and run [unpacked folder]/Bin/DSSimpleCaptureGL app in order to test. You should see the depth and color streams in separate windows.

Getting the nodelet to work

In the provided package, please run - \realsense_dist\[ver]\install_realsense_ros.sh

This will copy the nodelets and different configuration files to the /opt/ros/indigo.

You can find a sample nodelet in the folder under the sample folder. This shows the basic access to the camera streams.

Running the R200 nodelet

Simply type:

roslaunch realsense realsense_r200_launch.launch

This will launch the camera nodelet. You will see the camera lights up.

A quick check will be to run the stock image viewer to see the color image:

rosrun image_view image_view image:=/camera/color/image_raw

You can also open RVIZ and add sources based on topics.

Published Topic list:

Depth camera

      camera/depth/image_raw (sensor_msgs/Image)

          Raw image from device. Contains uint16 depths in mm.

       camera/depth/points (sensor_msgs/PointCloud2)

         Registered XYZRGB point cloud.

       camera/depth/uv (std_msgs/Float32MultiArray)

          UV mapping for depth image.

       camera/depth/image_textured (sensor_msgs/Image)

          Depth image colored with the RGB data.

Color camera

         camera/color/image_raw (sensor_msgs/Image)

          Color rectified image. RGB format.

 

R200 nodelet parameters:

cHeight(int, default: 480) 

                Specify the color camera height resolution.

cwidth(int, default: 640)

                Specify the color camera width resolution.

dHeight(int, default: 360)

                Specify the depth camera height resolution.

dwidth(int, default: 480)

                Specify the depth camera width resolution.

colorCamera(string, default: /dev/video1)

                Specify the color camera.

depthCamera(string, default: /dev/video2) 

                Specify the depth camera.

pcScale(float, default: 100)

                Specify a scale divisor for the point cloud. Use full to go from one measurement unit to another.

enableDepth(int, default: 1) 

                Specify if to enable or not the depth camera. 1 is true. 0 is false.

enableColor(int, default: 1) 

                Specify if to enable or not the color camera. 1 is true. 0 is false.

Known issues:

1.       After every run, there is a need to physically unplug and re-plug the camera. If not, an Error message Cannot determine firmware version! will appear when trying to access the camera.

2.        Check Changelog.txt for more info

 

 

 

 

Intel RealSense Technology F200 (ivcam) Integration to ROS (Robot Operating System)

Requirements:

Linux 12.04+

ROS Indigo

F200 (IVCAM) camera

* The ROS integration has been tested on a 64bit machine with Linux 14.04.01 and ROS Indigo.

Getting Started:

Getting the camera to work on Linux

Please make sure that you have a working camera and that the software stack is installed properly. This can be checked by connecting the camera (to a USB3 port) and running one of the provided samples.

If this does not work, you should first fix this issue before continuing with the ROS integration.

In the package folder you will find f200_install folder.

Simply run sh install.sh [path to install]. For example sh install.sh ~/ivcam

After installation is complete, in order to test :

1.       First build the provided sample in [installed folder]/ivcamsample/ by running the build script: sh [installed folder]/ivcamsample/build.sh

2.       Run [installed folder]/ivcamsample/ivcam_sample /dev/video1 /dev/video2 in the case you have an integrated webcam. If not, use video0 and video1.

The app just opens the camera and reads few frames. It does not have any GUI. The output should be something like that:

Known issues:

1.       First few frames may return error Failed to read image. Just ignore those.
